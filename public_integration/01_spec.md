# Спецификация: переход Nexus Looker на публичную интеграцию amoCRM

**Дата:** 2026-06-11 (v2 — после ревью агентом и codex, см. `03_plan_review_log.md`)
**Статус:** утверждена к разработке
**База:** ревью соответствия `work_directory/04_reviews/10_amocrm_requirements_compliance.md`
**Требования amoCRM:** https://www.amocrm.ru/developers/content/integrations/requirements

## 1. Цель

Превратить приватный виджет Nexus Looker (предпросмотр вложений в amoCRM) в **публичную
интеграцию маркетплейса amoCRM**, проходящую модерацию, с установкой в любой кабинет
без ручной настройки.

## 2. Утверждённые продуктовые решения (ответы владельца, 2026-06-11)

| Вопрос | Решение |
|---|---|
| Монетизация | **Бесплатно на старте.** Без «Внешней оплаты», цен и платёжки. Монетизация — позже отдельным релизом. |
| Рынки | **ru + en.** Русский маркетплейс + английская публикация (Kommo — отдельный маркетплейс, отдельный процесс подачи). Для en: материалы на английском, без упоминаний .ru-доменов, контакты поддержки на английском. Цены/PayPal не нужны (бесплатно). |
| Office-форматы (docx/pptx/xlsx/csv) | **Остаётся Microsoft Office viewer** через наш `/preview-host`. Обязательное раскрытие в описании и политике конфиденциальности. |
| Реквизиты в политике/поддержке | **Плейсхолдеры** `{{COMPANY_NAME}}`, `{{COMPANY_REQUISITES}}`, `{{SUPPORT_EMAIL}}` и т.п. Реальные значения вставляются перед подачей — разработку не блокирует. |

## 3. Рамки

**В рамках:**
- Код виджета и конвертера — устранение всех несоответствий требованиям (Н1–Н3 из ревью №10)
  и находок ревью плана (`03_plan_review_log.md`).
- Схема аутентификации конвертера, работающая для произвольного кабинета без настройки.
- Автотесты серверной auth/лимитов; build-guard от запрещённых паттернов в архиве.
- Материалы маркетплейса: политика конфиденциальности (ru/en) **с публикацией на сервере**,
  описания до/после установки (ru/en), инструкция, тур-изображения 1188×616, письмо в
  техподдержку amoCRM (риски М1-лента и М4-модалка).

**Вне рамок:**
- Монетизация, биллинг.
- Перевод Office-рендера на собственный конвертер.
- Регистрация юрлица, фактическая подача на модерацию (делает владелец).
- Изменение механики врезки в ленту (Plan B обсуждается с модерацией, решение за владельцем).

## 4. Изменения в виджете (клиент)

### W1. manifest.json
- Добавить обязательный блок `"settings": {}` (п. 3.1.2 требований). Валидность пустого
  блока проверяется загрузкой zip на venskons78; запасной вариант — однополевый settings.
- Версия → `0.2.0`.
- `support.link`/`support.email`: до подачи заменить на доменную почту/страницу поддержки
  (чек-лист); в коде — текущие значения.

### W2. Выпил vendorLoader (нарушение п. 3.2 + п. 3.1)
- `markdown-it` ставится из npm (**`dependencies` + обновлённый `package-lock.json`,
  проверка `npm ci`**) и **инлайнится в бандл** esbuild'ом.
- Удаляются: `src/vendorLoader.js`, `test/vendorLoader.test.js`, `vendor/`,
  копирование `vendor/` в `build.js`.
- Следствия: в архиве нет минифицированных файлов; нет `createElement('script')`;
  нет манипуляций `window.define`. Бандл по-прежнему `minify: false`.
- **Build-guard в `build.js`** (fail-fast после сборки): бандл не содержит
  `createElement("script"|'script')`, `eval(`, `new Function`, `alert(`, `confirm(`,
  живой ветки AMD (`define.amd` вне нашей обёртки — проверка smoke'ом); в dist/zip нет
  файлов `*.min.*` и каталога `vendor/`.

### W3. Убрать токен и settings-оверрайды из клиента
- `legacy.js`/`office.js` перестают слать `X-Source-Token` и читать
  `settings.converter_token` / `settings.converter_url` / `settings.preview_host_url` —
  эндпоинты становятся константами модулей. Любой токен в публичном zip — не секрет;
  скрытые оверрайды — лишняя конфигурационная поверхность.
- `settings`/`getSettings` удаляются из контракта рендереров (`modal.js`, `script.js`),
  если не остаётся потребителей. Контракт: `({ $, file, $body, params, loader, langs })`.

### W4. Локализация — полная (блокер en-публикации)
- Захардкоженные строки → i18n: `inject.js` тултип глазика (`'Предпросмотр'`) и дефолтное
  имя картинки (`'Изображение'`); `loader.js` юнит размера (`' МБ'`).
  Новые ключи: `eye.tooltip`, `image.default_name`, `units.mb`, `errors.rate_limited`.
- `Injector` получает переводы (проброс из `script.js`); тесты inject обновляются.
- en.json: парность ключей с ru.json; гильеметы «…» в en-текстах → прямые кавычки;
  без русского текста и .ru-доменов.
- Клиент различает 429: `legacy.js`/`office.js` мапят статус 429 → `langKey rate_limited`.

### W5. Чистка
- Удалить мёртвые стили `.nx-render-docx*`, `.nx-render-xlsx*`, `.nx-xlsx-*` из `style.css`
  (рендереры выпилены ранее).
- Обновить CLAUDE.md (vendorLoader исчезает) и README (давно устарел).

## 5. Изменения в конвертере (сервер)

### C1. Аутентификация: Origin-gate + токен, единый слой
- **`requireAuth`** (заменяет `requireToken`): пропуск, если
  (а) `Origin` матчит `ALLOWED_ORIGIN_PATTERN`, дефолт —
  **однометочный** поддомен кабинета, без вложенных уровней и порта:
  `^https://[a-z0-9][a-z0-9-]{0,62}\.(amocrm\.(ru|com)|kommo\.com)$`
  или (б) валидный `X-Source-Token` (`timingSafeEqual` — служебный путь, не для браузера).
- Результат кладётся в `req.auth`; **in-handler проверки** (`server.js:85,127`)
  переводятся на `req.auth` — без этого Origin-запросы получали бы 401 внутри хендлеров.
- Гейт стоит **ДО `express.raw`**. CORS отражает только Origin, прошедший паттерн;
  `Vary: Origin`; `X-Source-Token` удаляется из `Access-Control-Allow-Headers`.
- Понимание угрозы: Origin подделывается из curl. Защита от не-браузерного абуза —
  лимиты (C2) + существующие: p-limit, MAX_BYTES, timeout, allowlist расширений,
  kill process-tree. Риск принят владельцем (§8).

### C2. Лимиты — три эшелона
1. **Глобальный кап одновременных запросов с телом** (`MAX_INFLIGHT`, дефолт 16):
   счётчик inflate/deflate вокруг буферизации; превышение → 503 ДО `express.raw`.
   Закрывает «100×50 МБ параллельно = 5 ГБ RAM».
2. **Rate-limit по двум ключам**: фиксированное окно 60 с, отдельные счётчики
   `origin:<origin>` (`RATE_LIMIT_PER_MIN`, дефолт 30) и `ip:<req.ip>`
   (`RATE_LIMIT_PER_MIN_IP`, дефолт 60; `app.set('trust proxy', 1)` за nginx).
   Двойной ключ закрывает ротацию поддоменов. Map ограничена (`MAX_RATE_KEYS`,
   дефолт 5000), очистка по границе окна; переполнение после очистки = глобальная
   перегрузка → 429 для новых ключей. OPTIONS не расходует лимит. Превышение → 429.
3. nginx `limit_req` per-IP + `client_max_body_size` — второй эшелон (проверяется
   на деплое как явный пункт).

### C3. /preview-host: сжатие окна экспозиции
- `PREVIEW_TTL_MS` дефолт **5 мин** (было 15; Microsoft забирает файл за секунды).
- nginx для `/preview/`: `Cache-Control: no-store`, `X-Robots-Tag: noindex`,
  `X-Content-Type-Options: nosniff` (+ существующие autoindex off, alias с trailing slash).
- Лимит `/preview-host` может быть строже (`RATE_LIMIT_PREVIEW_PER_MIN`, дефолт = общий).

### C4. Тестируемость и конфигурация
- `server.js` разделяется: **`converter/app.js`** — фабрика `createApp({ convert })`
  (DI тяжёлой конвертации), `server.js` — entry (listen + TTL-уборка).
- Автотесты (vitest + supertest из корня): матрица auth (Origin валидный/невалидный/
  вложенный поддомен/порт/http/нет; токен валидный/нет), 429 по origin и по ip,
  глобальный 503-кап, CORS на POST/OPTIONS/ошибке 413, отсутствие буферизации без auth.
- env: `ALLOWED_ORIGIN_PATTERN`, `RATE_LIMIT_PER_MIN`, `RATE_LIMIT_PER_MIN_IP`,
  `MAX_RATE_KEYS`, `MAX_INFLIGHT`, `PREVIEW_TTL_MS` — в `converter/deploy/` примерах.
- Харднинг LibreOffice-контейнера (пункт деплоя): non-root уже есть; проверить
  отсутствие лишнего egress и read-only где возможно.

## 7. Безопасность — инварианты

1. Auth-gate (Origin-паттерн или токен) **до** буферизации тела; глобальный кап inflight.
2. `timingSafeEqual` для токена; длина сравнивается до содержимого.
3. p-limit на тяжёлые операции; kill process-tree LibreOffice по таймауту/abort.
4. Allowlist расширений на входе; путь записи — `randomUUID()`; TTL-очистка preview.
5. Рендер недоверенного контента — только через безопасные синки. Единственный
   допустимый `innerHTML` — выход markdown-it с `html:false` (raw HTML экранируется,
   опасные ссылки фильтруются); svg — только `<img blob:>`; остальное — `.text()`/DOM API.
6. Никаких секретов в публичном zip.
7. `loader.js`: `credentials: 'same-origin'` (не трогать).
8. Build-guard в `build.js` — сборка падает при запрещённых паттернах (W2).

## 8. Риски

| Риск | Вероятность | Митигация / статус |
|---|---|---|
| **М1: модерация отклонит врезку глазиков в ленту** (п. 2.2) | Высокая | Письмо в техподдержку amoCRM ДО подачи (включая модалку-оверлей). Единственный нерешаемый кодом риск; изменение механики — решение владельца после ответа amo. |
| Origin-spoofing не-браузерными клиентами | **Принят владельцем** | Трёхэшелонные лимиты (C2), форматы/размеры ограничены, TTL. Худший случай — чужое использование бесплатного конвертера в рамках лимитов. |
| **Репутация домена**: `/preview-host` как анонимный краткоживущий файлохостинг → раздача малвари → Safe Browsing/блок-листы → отказ Office-предпросмотра у всех | Средняя | TTL 5 мин, no-store/noindex/nosniff, строгий лимит, мониторинг объёма публикаций. Остаточный риск принят (следствие выбора «Microsoft viewer + бесплатно»). |
| Нагрузка на конвертер (бесплатно для всех) | Средняя | C2-лимиты; p-limit; мониторинг; монетизация позже. |
| Microsoft viewer: недоступность/лимит 15 МБ | Низкая | Уже обработано (langKey-ошибки). |
| en-модерация (Kommo): отдельный процесс | Средняя | Отдельный раздел чек-листа; en-материалы без .ru; уточнить требования формы подачи Kommo до подачи. |

## 9. Критерии приёмки

1. `npm ci && npm test` зелёные (включая новые тесты конвертера); `npm run build`
   собирает zip; **build-guard активен** (искусственная порча — сборка падает).
2. `manifest.json`: `widget` + `locations` + `settings`; версия 0.2.0; zip загружается
   на venskons78 без ошибок валидатора.
3. Виджет не содержит токена/оверрайдов; Office/legacy предпросмотр работает через
   Origin-схему (smoke на venskons78 с обновлённой widget-host статикой).
4. Матрица конвертера (автотесты + curl на проде): Origin `https://x.amocrm.ru` без
   токена → 200; `https://a.b.amocrm.ru`, `https://evil.com`, без Origin → 401;
   токен без Origin → 200; превышение per-origin и per-ip → 429; inflight-кап → 503;
   тело не буферизуется без auth; 413 с CORS-заголовками.
5. Материалы §6 существуют; политика/поддержка доступны по публичным https-URL;
   en-версии без русского текста и .ru-доменов.
6. Кодировки: UTF-8 без BOM, LF (все новые/изменённые текстовые файлы).
7. Security-review без HIGH/MEDIUM, **исключая риски, явно принятые в §8**.

## 10. Открытые вопросы (не блокируют)

- Реальные реквизиты и доменная почта поддержки — владелец, перед подачей.
- Ответ amoCRM на письмо (лента + модалка) — внешняя зависимость.
- Проверка реальных Origin кабинетов amoCRM/Kommo (включая поддомены com) — чек-лист.
- Валидность `"settings": {}` — проверка загрузкой zip (запасной вариант готов).
