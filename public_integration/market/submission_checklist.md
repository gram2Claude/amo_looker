# Чек-лист подачи Nexus Looker в маркетплейс

## A. Перед подачей — заполнить плейсхолдеры (владелец)

| Плейсхолдер | Где | Что вставить |
|---|---|---|
| `{{COMPANY_NAME}}` | site/privacy_*.html, support_*.html | Название юрлица/ИП |
| `{{COMPANY_REQUISITES}}` | site/privacy_*.html | ИНН/ОГРН(ИП), адрес |
| `{{SUPPORT_EMAIL}}` | site/*, market/* | **Доменная** почта поддержки (не gmail — модерация может счесть несолидным; завести support@naithon.one или аналог) |
| `{{POLICY_DATE}}` | site/privacy_*.html | Дата редакции политики |
| `{{DEVELOPER_NAME}}` | moderation_request_ru.md | Имя/название разработчика |
| `support.link` / `support.email` | manifest.json | https://nexus-oko.naithon.one/legal/support_ru.html + доменная почта |

## B. Внешние шаги (владелец)

- [ ] **Отправить письмо** `moderation_request_ru.md` в техподдержку разработчиков amoCRM
      и дождаться ответа про врезку в ленту и модалку (главный риск отказа — п. 2.2).
- [ ] Завести доменную почту поддержки.
- [ ] Реквизиты юрлица для политики.

## C. Технические проверки (можно силами Claude)

- [ ] zip 0.2.0 загружается на venskons78 без ошибок валидатора
      (в т.ч. проверка, что `"settings": {}` принимается; если нет — заменить на
      однополевый блок с текстовым полем, описано в спеке W1).
- [ ] Smoke на venskons78 через букмарклет: pdf / jpg / svg / md / docx / .doc / csv.
- [ ] curl-матрица auth на проде (Origin / токен / 429 / 503 / 413+CORS).
- [ ] Политика и поддержка открываются: https://nexus-oko.naithon.one/legal/privacy_ru.html
      (+ _en, support_*).
- [ ] Сверить реальный Origin кабинетов: открыть DevTools в кабинете amocrm.ru
      и (при наличии) kommo.com, убедиться что Origin запросов матчит
      `ALLOWED_ORIGIN_PATTERN` (однометочный поддомен).

## D. Изображения

- [ ] Иконка 400×272 (есть: images/logo_main.png), логотип 108×108 (logo_small.png),
      130×100 (logo.png) — все не анимированные.
- [ ] **Тур 1188×616, минимум 3 шт на язык.** В `market/tour/` лежат сгенерированные
      макеты-черновики. ПЕРЕД ПОДАЧЕЙ заменить на реальные скриншоты из кабинета
      venskons78 (букмарклет → открыть предпросмотр → скриншот): модерация может
      отклонить изображения, не соответствующие реальному интерфейсу.
