Здравствуйте!

Готовим к подаче в маркетплейс публичную интеграцию **Nexus Looker** — предпросмотр
файлов-вложений без скачивания. Перед подачей хотим согласовать две детали
UI-механики, чтобы корректно соответствовать разделу «Изменение интерфейса»
требований к интеграциям.

**1. Кнопка предпросмотра рядом с вложениями в ленте.**
Виджет (locations: lcard-1, ccard-1, comcard-1) добавляет небольшую круглую кнопку
(22×22 px) рядом со строками файлов-вложений в ленте примечаний — после имени файла,
не поверх существующих элементов. Технически: MutationObserver на контейнере ленты,
кнопка дописывается в конец контейнера строки вложения. Кнопка:
- ничего не перекрывает и не смещает штатные элементы управления;
- не изменяет и не перехватывает штатное поведение ссылок (скачивание по клику на имя
  файла работает как раньше);
- полностью исчезает при отключении интеграции (стандартное отключение в настройках).

Вопрос: допустима ли такая врезка для публичной интеграции? Если для этого сценария
предусмотрен другой рекомендованный механизм (документированная область/виджетная
зона для действий с вложениями) — подскажите, пожалуйста, какой.

**2. Модальное окно предпросмотра.**
По клику на кнопку открывается модальное окно (оверлей затемнения + окно по центру,
до 90% вьюпорта). Окно:
- открывается ТОЛЬКО по явному действию пользователя;
- закрывается крестиком, кликом по затемнению и клавишей Esc;
- не появляется самопроизвольно и не блокирует работу фоном.

Вопрос: достаточно ли этого для соответствия требованию «не перекрывать элементы
управления amoCRM», учитывая что перекрытие — временное и инициировано пользователем
(аналогично штатным модальным окнам amoCRM)?

Краткая справка об интеграции: бесплатная; настроек не требует; данные CRM не читает
и не изменяет (виджет работает только с файлом, который пользователь открыл);
для отображения docx/pptx/xlsx/csv используется Microsoft Office Viewer (раскрыто
в описании и политике конфиденциальности); остальные форматы обрабатываются в браузере.
