Nexus Looker Widget Privacy Policy

Last updated: {{POLICY_DATE}}

This policy describes what data the Nexus Looker widget (the "Widget") processes and what happens to it. The Widget is developed and maintained by {{COMPANY_NAME}} (the "Developer").

1. What the Widget does

The Widget adds a preview button next to file attachments in the CRM interface and shows the file content in a modal window without downloading.

2. What data is processed

  • Attachment file content — only for files the user explicitly clicked the preview button for. The Widget never processes files in the background.
  • Technical request data sent to the Developer's server (IP address, account address in the Origin header) — to the minimum extent required to protect the service from abuse (rate limiting).

The Widget does not collect or store CRM customers' personal data and does not read leads, contacts, conversations or any CRM data other than the file the user opened.

3. Where files go during preview

  • PDF, images (including SVG), plain text, Markdown — processed entirely in the user's browser. Never transmitted anywhere.
  • Legacy Office formats (.doc, .xls, .ppt, .rtf, .odt) — sent to the Developer's server, converted to PDF and immediately returned to the browser. The file is not stored; file contents and names are not logged.
  • Modern Office formats (.docx, .pptx, .xlsx, .csv) — rendered with Microsoft Office Online Viewer. The file is sent to the Developer's server and temporarily (no longer than 5 minutes) published at an unguessable one-time URL from which the Microsoft service fetches it. After that the file is deleted automatically. Microsoft's processing is governed by the Microsoft Privacy Statement.

4. Storage and logging

The Developer's server does not keep files longer than needed for conversion or preview (at most 5 minutes for Office formats). File contents are never logged. Server logs are limited to data required for reliability and abuse protection: an internal request id, file type (extension) and size, processing stage durations, result code and request country. File names, file contents and IP addresses are never written to logs; logs are rotated automatically (about 30 days).

5. Third parties

The only third party that receives file content is Microsoft (only for .docx, .pptx, .xlsx, .csv — see section 3). Data is never sold or shared with anyone else.
