## Сводка находок и решений

| # | Находка (источник) | Severity | Решение |
|---|---|---|---|
| 1 | In-handler проверки токена (`server.js:85,127`) останутся 401 для Origin-запросов, если менять только middleware (агент M3, codex CRITICAL) | CRIT | Единый `requireAuth` → `req.auth`; in-handler проверки переводятся на `req.auth` (defense-in-depth сохраняется) |
| 2 | Origin-gate ≠ аутентификация: буферизация тела фактически анонимная; rate-limit per-Origin обходится ротацией поддоменов; map без политики вытеснения (агент H1, codex CRITICAL) | CRIT | Глобальный кап одновременных запросов с телом (503 до `express.raw`); лимитер по ДВУМ ключам — origin И ip (`trust proxy` за nginx); map с капом и очисткой по окну; при переполнении map — 429 новым ключам (= глобальная перегрузка) |
| 3 | Origin-regex слишком широкий; реальные форматы доменов amoCRM/Kommo не проверены (codex CRITICAL) | CRIT | Однометочный поддомен: `^https://[a-z0-9][a-z0-9-]{0,62}\.(amocrm\.(ru|com)|kommo\.com)$` (без вложенных поддоменов, без порта); негативные тесты; пункт чек-листа «сверить с реальными origin» |
| 4 | `/preview-host` = анонимный 15-мин файлохостинг → раздача малвари с нашего домена, риск Safe Browsing/блок-листов → смерть Office-предпросмотра для всех (агент H2) | HIGH | TTL ↓ до 5 мин (env); nginx `/preview/`: `Cache-Control: no-store`, `X-Robots-Tag: noindex`, `X-Content-Type-Options: nosniff`; лимит на `/preview-host` строже; харднинг LibreOffice-контейнера — пункт деплоя; риск репутации домена внесён в §8 спеки как принятый с митигациями |

| 6 | Клиент не отличает 429 → пользователь видит «сервис недоступен» (codex HIGH) | HIGH | `legacy.js`/`office.js`: статус 429 → `langKey rate_limited`; ключ в ru/en |
| 7 | Политика/страница поддержки нигде не хостятся; критерий приёмки удовлетворяется без публичных URL (агент M1) | MED | Этап 4: выложить privacy/support на nginx; критерий приёмки — «URL открываются по https» |
| 8 | Этап 4.3 тестировал бы старый бандл: деплой widget-host статики не запланирован (агент M2) | MED | Этап 4: пересобрать и выложить статику; boot.js без токена (тест целевого Origin-пути) |
| 9 | Sequencing: zip без токена не работает со старым прод-конвертером (codex MED) | MED | Порядок: деплой сервера (обратносовместимого) → потом сборка/раскатка виджета. Внесено в план |
