# Спека: Nexus Looker (amo_looker)

**Дата:** 2026-06-10. Источники: handoff Михаила (`reuse/docs/handoff_plan.md`, `handoff_README.md`), DOM-разведка (`work_directory/01_specs/01_dom_recon_amocrm.md`), инвентарь (`reuse/INVENTORY.md`).

## Цель

Заменить платный CatCode Looker собственным приватным amoCRM-виджетом: предпросмотр прикреплённых файлов в карточках (сделка/контакт/компания) без скачивания — «глазик» рядом с файлом → модалка с рендером.

## Deliverables

1. **Виджет** (zip, ставится в amoCRM руками): инъекция глазика в ленту, модалка, рендереры PDF/изображения/текст/DOCX/XLSX/legacy.
2. **Конвертер** `amo-preview-converter` (Node + LibreOffice headless, docker) на сервере `95.216.44.25` (Hetzner) под `https://nexus-oko.naithon.one` — для .doc/.xls/.ppt/.pptx/.rtf/.odt.
3. **Релиз на прод** = кабинет **venskons78.amocrm.ru** (решение пользователя 2026-06-11: этот кабинет считаем продом). CatCode в нём НЕ установлен (проверено) — пункты про параллельный прогон с CatCode и его отключение неприменимы в этом периметре; если CatCode где-то ещё (кабинет Михаила) — отдельный вопрос.

## Критерии приёмки (e2e, из handoff плана — 10 проверок)

1. DOCX рендерится в модалке без скачивания; 2. XLSX; 3. PDF; 4. JPG/PNG; 5. legacy .doc/.xls через конвертер → PDF; 6. TXT/CSV как `<pre>`; 7. XLSX 20 МБ → корректное сообщение «слишком большой»; 8. неподдерживаемый формат → fallback «Скачать»; 9. 10 переключений карточек подряд — без утечек observer; 10. удаление виджета — глазики исчезают, DOM чистый.

## Ограничения и зависимости

- **Прод и разработка — один кабинет `venskons78.amocrm.ru`** (решение 2026-06-11). Доступ есть, тестовая сделка 3177663. SSH/DNS — закрыты своим сервером (95.216.44.25 / nexus-oko.naithon.one). Скриншот CatCode и право на чужой прод-кабинет — не нужны (CatCode в venskons78 нет).
- ⚠️ **БЛОКЕР прода:** на venskons78 лоадер amoCRM НЕ загружает виджет автоматически (похоже на «технический аккаунт», `amohash` пуст) — виджет поднимался только ручным `require` в консоли. Для реального использования виджет обязан грузиться сам при открытии карточки. Разбирается отдельно (см. `04_reviews/08_autoload_investigation.md`).
- amoCRM DOM может меняться без объявления — селекторы defensive, feature-flag отключения.
- LibreOffice-конвертер обрабатывает данные клиентов: non-root контейнер, без логирования содержимого, shared secret, лимиты.
- amoCRM не имеет авто-обновления приватных виджетов — каждый релиз руками через zip.

## Известные технические риски (из handoff + разведки)

1. PDF-рендерер на `<iframe src=href>` может упереться в `Content-Disposition: attachment` → скачивание вместо рендера; запасной план — pdf.js в vendor.
2. `<img src>` для картинок — те же редиректы; вероятно ок, проверить.
3. Большие XLSX блокируют UI (SheetJS синхронный) — лимит 10 МБ уже в коде.
4. LibreOffice RCE-история — изоляция контейнера обязательна.
5. Конфликт двух глазиков на проде в переходный период — разнести визуально.
6. В 4 рендерерах `credentials:'include'` — обязательный фикс на `'same-origin'` до любого QA.

## Блоки работ (верхнеуровнево)
