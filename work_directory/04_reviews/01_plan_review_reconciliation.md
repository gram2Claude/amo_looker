## Приложение A — полный текст ревью субагента

(вердикт: «план структурно зрелый, но публиковать рано: 5 критичных дыр»)

### Критичные
1. Секрет конвертера утечёт через публичный репозиторий (repo_visibility: public + токен в коде/zip) → токен только из env, done_criteria «в git-истории и releases/*.zip секрета нет»; признать в рисках, что токен в клиентском виджете в принципе извлекаем.
2. T12 (доступы) в календаре на 23.06 — в день, когда уже нужен результат → перенести в E1S1 как задачу-ожидание; T9 тоже зависит от Михаила (образцы КП) — запросить вместе.
3. Кейс 8 чек-листа невыполним: inject.js:80 `if (!canPreview(meta)) return;` — для .heic глазик не врезается, модалки с fallback не будет → определить поведение в T17 с правкой inject.js.
4. Feature-flag отключения заявлен в спеке, отсутствует в задачах → добавить kill-switch или вычеркнуть из спеки.
5. Тесты: ноль автоматизации и ноль решения → добавить задачу (unit fileUtils + Injector на jsdom + smoke конвертера) или явное решение «v1 без автотестов».

### Важные (выборочно)
- inject.js:36 — setTimeout-ретрай не отменяется в stop() (re-arm observer после destroy).
- fonts-dejavu недостаточно для русских документов → fonts-liberation, carlito, caladea.
- p-limit(4) с одним профилем LO даст блокировки → -env:UserInstallation per request.
- OPTIONS-preflight придёт без токена → отвечать 204 без авторизации, ACAH.
- QA чата/писем может быть невозможен в venskons78 → подготовка интеграций или перенос в прод-прогон.
- T8: blob→iframe (приём из legacy.js:24-26) — план A2; pdf.js тянет worker и ~2МБ.
- Мониторинг прод-прогона без механизма → /health, измеримые критерии T20.
- Цикл фикстур T9/T10↔T11.
- Docker локально не проверен (подтверждено: отсутствует).
- Граница T6/T17 по i18n-ошибкам размыта (modal.js:48-50 печатает сырой err.message; захардкоженные строки в 4 рендерерах мимо i18n).

### Оценка часов
T1 переоценен (4→~2), T4 переоценен (4→~3), T17 недооценен (3→4+), T13 на грани, +3-4 ч отсутствующих работ. Сумма 60.5 адекватна при перераспределении.
