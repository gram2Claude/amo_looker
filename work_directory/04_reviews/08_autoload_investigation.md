# Расследование: виджет не загружается автоматически на venskons78

**Дата:** 2026-06-11. Вопрос: почему на venskons78 (теперь = «прод» по решению пользователя) виджет не появляется сам при открытии карточки — глазики были только при ручной инъекции `require` в консоли.

## Что установлено (факты)

1. **Аккаунт venskons78 — «Технический аккаунт»** (бейдж в шапке amoМаркет на всех страницах), создан 10.06.2026 (`created_at=1781156410`), № клиента 33093102, владелец «Михаил».
2. **`amohash` пуст** в ответе `/ajax/widgets/list` — на всех проверках, при любом состоянии интеграции. На рабочих аккаунтах amoCRM это поле непустое (хэш для подписи виджет-вызовов фронтом).
3. **`widgets/list` возвращает наш виджет БЕЗ `path`/`version`** (только `init_once` + `integration_supported_features`) — фронт-лоадеру нечего грузить (нет URL скрипта).
4. **Скрипт виджета с CDN** (`widgets.amocrm.ru/upl/<code>/widget/script.js`) фронтом автоматически НЕ запрашивается (`performance` — 0 запросов), `AMOCRM.widgets.list` пуст, `.nx-eye` = 0 без ручной инъекции.
5. **Ручная инъекция работает** — `require([...script.js])` + `render/init` даёт 6/6 глазиков, docx/xlsx/pdf рендерятся. **Код виджета доказанно исправен.**
6. Побочно найдено: при **перезаливке zip** («Загрузить новый») и переименовании интеграция уходит в статус **«Отключено»** → нужен повторный OAuth-грант. После гранта виджет снова в `widgets/list`, но п.3–4 не меняются.

## Вывод (корневая причина)

Кабинет venskons78 — **технический аккаунт разработчика amoCRM**. В таком аккаунте фронт-лоадер виджетов **не исполняет виджеты в боевом UI** (пустой `amohash`, нет `path` в `widgets/list`) — это режим для разработки/загрузки архива, а не для «живой» работы виджета. Совокупность признаков (бейдж «Технический аккаунт» + пустой amohash + отсутствие path + свежая регистрация) указывает на это однозначно; публичный виджет на таком аккаунте загрузился бы так же — никак (тот же лоадер, тот же пустой amohash).

**Это НЕ дефект нашего кода** — код виджета работает (ручная инъекция доказала весь путь: инъекция → модалка → рендереры → конвертер).

## Следствие для прода

Конфликт с решением «venskons78 = прод»: на техническом аккаунте виджет нельзя «включить» для реальных пользователей — у них он сам не загрузится. Для настоящего прода нужен **обычный рабочий аккаунт amoCRM** (не технический, с активной подпиской/триалом), куда ставится та же приватная интеграция — там виджет грузится автоматически.

### Варианты (решение за пользователем/Михаилом)
1. **Перевести venskons78 из технического в обычный** — если это возможно в его настройках/у поддержки amoCRM (технический аккаунт обычно нельзя «переключить», создаётся отдельный рабочий).
2. **Использовать обычный рабочий кабинет** как прод (напр. боевой кабинет Михаила `toolkeeper.amocrm.ru` или новый рабочий аккаунт) — поставить туда приватную интеграцию (процедура отработана, ~5 мин).
3. **Оставить venskons78 для разработки/демо** через ручную инъекцию (как сейчас) — годится для показа, но не для боевого использования сотрудниками.

### Финальное подтверждение
100%-ную точку поставит установка нашей приватной интеграции на ОБЫЧНЫЙ рабочий аккаунт: ожидаем непустой `amohash`, `path`/`version` в `widgets/list` и авто-появление глазиков. До этого вывод — по совокупности косвенных (высокая уверенность).

## Операционная заметка
После каждого обновления zip интеграция деактивируется → повторить OAuth-грант (`https://www.amocrm.ru/oauth?client_id=<id>&state=...&mode=post_message` → выбрать аккаунт → Разрешить), иначе виджет пропадает из `widgets/list`.
