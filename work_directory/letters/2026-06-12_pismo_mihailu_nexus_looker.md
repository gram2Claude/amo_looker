Как только определишься — мы обновим описания и форму подачи. Остальные шаги владельца (реквизиты юрлица для политики конфиденциальности, доменная почта поддержки, скриншоты) перечислены в `public_integration/market/submission_checklist.md`.

## 3. Соответствие требованиям amoCRM

Мы прогнали виджет по требованиям amoCRM к публичным интеграциям и внесли все необходимые правки:

- **Все зависимости встроены в бандл** — никакой динамической догрузки скриптов (требование п. 3.2). Прежний механизм догрузки vendor-библиотек выпилен полностью.
- В коде **нет** `eval`, `new Function`, `alert`, `confirm`, минифицированных файлов и папки `vendor/`.
- **Секретов и токенов в клиентском коде нет** — авторизация конвертера переведена на проверку Origin кабинета (`*.amocrm.ru`, `*.kommo.com`), работает у любого клиента без настройки.
- Сборка защищена **build-guard'ом**: при появлении любого запрещённого паттерна в бандле сборка падает с ошибкой — регресс по требованиям модерации невозможен незаметно.
- Архив уже устанавливался в кабинет и **проходил валидатор amoCRM без ошибок**.

## 4. Безопасность и работоспособность

**Безопасность** — три независимых цикла ревью (внутренний агент + codex + отдельное security-ревью), все находки исправлены:

- Весь рендер недоверенного содержимого файлов идёт через безопасные методы: markdown рендерится с запретом HTML, SVG показывается как картинка (скрипты внутри не исполняются), PDF и изображения — через локальные blob-URL.
- Конвертер закрыт эшелоном лимитов: глобальный кап одновременных запросов, rate-limit по Origin и IP, лимиты nginx; найденные на ревью обходы (через OPTIONS-запросы и накопление тел в очереди) устранены.
- Файлы на конвертере живут максимум 5 минут и удаляются автоматически.
- Приватность по форматам: PDF, картинки, SVG, текст, markdown и старые .doc/.xls/.ppt **не покидают наш контур** (рендер локально или на нашем сервере); docx/xlsx/pptx/csv открываются через официальный Microsoft Office viewer — это осознанное решение, отражено в политике конфиденциальности.

**Работоспособность** — проверено 12.06.2026:

- 56/56 автотестов (включая тесты врезки на реальной разметке ленты amoCRM и тесты авторизации/лимитов конвертера);
- живой e2e в кабинете amoCRM на **13 форматах**: pdf, jpg, webp, svg, md, txt, json, docx, xlsx, pptx, csv, .doc, .xls — все открываются корректно, включая реальный «боевой» .doc с кириллицей в имени;
- криптографическая сверка: sha256-хэши кода, который исполнялся в тестах, побайтово совпадают с файлами внутри передаваемого zip — тестировали ровно то, что сдаём.

### Задание: независимая проверка виджета Nexus Looker

Тебе нужно проверить виджет для amoCRM перед подачей на модерацию. Три части: проверка кода, проверка серверной части, живой тест в кабинете.

**Потребуется**: Node.js 18+, git, curl. Для части В — браузер, в котором пользователь залогинен в свой кабинет amoCRM.

#### Часть А. Код: тесты, сборка, сверка с продом

```bash
git clone https://github.com/gram2Claude/amo_looker.git
cd amo_looker
npm ci
npm test
# ожидаемо: 4 файла тестов, 56 passed, 0 failed

npm run build
# ожидаемо: сборка без ошибок, создан releases/nexus-looker-0.2.0.zip (~79 КБ)
# build-guard внутри сборки сам проверит отсутствие запрещённых amoCRM паттернов
```

Сверка, что продакшен-сервер отдаёт ровно тот код, что в архиве (Linux/macOS/Git Bash):

```bash
unzip -o releases/nexus-looker-0.2.0.zip script.js style.css -d /tmp/zipcheck
curl -s -o /tmp/served_script.js https://nexus-oko.naithon.one/widget/script.js
curl -s -o /tmp/served_style.css https://nexus-oko.naithon.one/widget/style.css
sha256sum /tmp/served_script.js /tmp/zipcheck/script.js
sha256sum /tmp/served_style.css /tmp/zipcheck/style.css
# ожидаемо: хэши попарно СОВПАДАЮТ
```

(на Windows вместо `sha256sum` — `Get-FileHash -Algorithm SHA256 <файл>`)

#### Часть Б. Серверная часть: авторизация и доступность

```bash
# страницы политики и поддержки отдаются:
curl -s -o /dev/null -w "%{http_code}\n" https://nexus-oko.naithon.one/legal/privacy_ru.html   # 200
curl -s -o /dev/null -w "%{http_code}\n" https://nexus-oko.naithon.one/legal/support_ru.html   # 200

# конвертер отклоняет запросы без Origin кабинета:
curl -s -o /dev/null -w "%{http_code}\n" -X POST https://nexus-oko.naithon.one/convert          # 401
curl -s -o /dev/null -w "%{http_code}\n" -X POST -H "Origin: https://evil.example.com" \
  https://nexus-oko.naithon.one/convert                                                          # 401

# конвертер пускает запрос с Origin кабинета amoCRM (415 = авторизация пройдена,
# отбой на валидации содержимого — это правильное поведение для пустого тела):
curl -s -o /dev/null -w "%{http_code}\n" -X POST -H "Origin: https://toolkeeper.amocrm.ru" \
  -H "Content-Type: application/octet-stream" --data-binary "x" \
  https://nexus-oko.naithon.one/convert                                                          # 415
```

#### Часть В. Живой тест в кабинете amoCRM

Важный контекст: до публикации в маркетплейсе amoCRM не исполняет виджет в кабинете автоматически, поэтому для проверки используется закладка-активатор. Часть А (сверка хэшей) доказывает, что закладка загружает ровно тот код, который лежит в передаваемом zip.

1. Создай в браузере закладку с таким адресом (одной строкой):

```
javascript:(function(){var s=document.createElement('script');s.src='https://nexus-oko.naithon.one/widget/boot.js?'+Date.now();document.body.appendChild(s);})();void(0);
```

2. Залогинься в кабинет amoCRM, открой любую сделку, в ленте которой есть вложения (pdf, docx, картинки — что угодно). Если такой нет — прикрепи пару файлов примечанием.
3. Нажми закладку. Рядом с каждым вложением в ленте появится синяя кнопка-«глазик» (видна при наведении на строку файла).
4. Кликни глазик — файл откроется в модальном окне предпросмотра, без скачивания.

Ожидаемое поведение по форматам:
- pdf, картинки (jpg/png/webp/svg), txt, md, json — открываются мгновенно, рендер локальный;
- docx, xlsx, pptx, csv — открываются через Microsoft Office viewer, задержка 2–5 секунд;
- старые .doc/.xls/.ppt — конвертируются на нашем сервере в PDF, задержка 5–15 секунд;
- у файла, добавленного в ленту прямо сейчас, глазик появляется сам, без повторного нажатия закладки;
- после перезагрузки страницы закладку нужно нажать заново (в опубликованной версии виджет будет грузиться автоматически).
